Q: Powershell 连接SSH 需要指定密钥文件

使用Win10下自带OpenSSH命令，配置好客户端id_rsa和服务端id_rsa.pub，键入如下代码连接：

ssh username@x.x.x.x

失败

Premit denied(Public key)

必须用-i参数指定

ssh username@x.x.x.x -i /path/id_rsa

Success!

在某些时候比如，rsync调用ssh时，无法附加-i参数，且从理论上来说是无需指定的。

A: 用config文件指定

其实以前刚开始接触ssh的时候就有这个问题，就是客户端如何给不同的ip和用户分配密钥。

当时那博主语焉不详，想来自己估计也不知道。

/$userpath$/.ssh/是ssh调用密钥的默认路径，这个文件夹下如果从未进行ssh连接的情况下只有你刚复制进去的id_rsa密钥。尝试连接后，会自动生成known_host文件储存服务端指纹。

在此文件夹创建config文件。注：无后缀名文件。

输入以下配置：

Host ali

​ HostName x.x.x.x

​ Port 22

​ User username

​ IdentityFile ~/.ssh/id_rsa

配置完后，ssh会自动根据config文件在连接时自动指定密钥。

Q: id_rsa权限问题

win10 ssh连接时出现：

@@@

WARNING

@@@

id_rsa TOO OPEN!

修改id_rsa权限为仅限拥有者完全控制，去除所有其他组合用户权限，并且当前执行ssh的用户是id_rsa文件拥有者。

chmod 600 ./id_rsa

用cWrsync的client端bin目录下自带ssh执行时，即使修改了权限，有时还是会出现too open错误，将/$username$/.ssh/整个文件夹复制入cwrsync目录下的对应文件夹即可。具体原因不明，猜测可能是调用自带ssh时候权限继承有关。

End
